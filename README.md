# 小型超市 3D 仿真（MVP）

这是一个可直接运行的 Web 版 3D 小型超市原型，包含：

- 货架、冰柜、储物架、前台终端、取货车
- 基础场景与可视化
- 基础运营系统（开店/关店、顾客进店、取货、结算、资金与销量）
- 资源调控器（动态调节资金、库存、顾客数、客流间隔、售价）
- 第一人称店长视角（可在店内行走并交互）
- RFID 仓配流程场景（仓储后仓、配送卡车、手持 Reader、LCD 终端）
- 置办设施（点击地面放置）
- 选中后移动、旋转、删除
- 给选中货架补货
- 布局保存/读取（本地 `localStorage`）

## 运行方式

推荐用本地静态服务器运行（避免浏览器跨域限制）。
项目通过 CDN 加载 Three.js，请确保网络可访问 `cdn.jsdelivr.net`。

### 方式 1：Python

```bash
python -m http.server 8000
```

打开：`http://localhost:8000`

### 方式 2：Node（任选）

```bash
npx serve .
```

## 操作说明

- 默认会生成一套可运行的超市基础布局
- 默认相机为第一人称，点击场景后锁定鼠标
- 可用 `切换到俯视相机` 在第一人称/俯视之间切换
- 点击 `开店` 后，顾客会自动进店 -> 去货架拿货 -> 到前台结算 -> 离店
- `全货架补货（$20）`：消耗资金并增加仓库存货，再给货架补货
- `资源调控器`：实时修改资金、库存、顾客数量、客流间隔和商品售价
- 仓库侧流程：先拿 `RF标签器`，选中货品按 `T` 绑定 RFID；再切换手持 Reader 对货品按 `E` 录入系统
- 配货仓货品默认摆放在货架上，可取下放到地面后执行贴标与扫码
- 配货仓货架三层会补满货品，且水果/生鲜/烘焙/冷饮/综合分别放在对应类别货架
- 仓库侧手持Reader录入后，货品会自动装车到车厢上方（便于继续交互）
- 对准“仓配系统 LCD 终端”按 `E` 进入系统界面，可登记配送单/装车/发运/确认收货
- 门店侧流程：货车送达后仓后，手持 Reader 对货品按 `E` 扫描比对清单
- 门店后仓新增手持 Reader 工具台用于收货比对流程
- 门店后仓 LCD 固定在后仓后墙区域（不放在店内），且朝向独立避免与配货仓同向镜像
- 两个仓库都新增大屏LCD看板：配货仓显示订货需求完成进度，门店仓显示收货比对正确/异常进度
- 两块LCD为独立实体，分别固定在配货仓与门店后仓，显示内容不互相同步
- 门店扫码通过后，货品会自动归位到对应类型后仓货架，供后续补货使用
- 在 LCD 里执行“按扫描结果更新门店库存”，通过件入库，异常件会记录通知
- 场景布局：上方是独立“配货仓库”建筑；下方是门店，门店后部为“门店后仓”；两栋建筑前后门均为敞开通道
- 左侧点击设施类型进入放置模式
- 在 3D 场景里点击地面放置
- 鼠标左键点击设施可选中
- 已选中对象名称会显示在“当前状态”面板（便于区分 Reader/LCD/货品/卡车）
- 已选中对象名称会悬浮显示在场景中该物体顶部
- 左侧“使用说明”会根据当前选中对象动态提示操作方法
- 对准“RF标签器”按 `E` 拿标签工具；对准“手持RF Reader”按 `E` 切换Reader工具
- 贴标与扫货均要求先拿对应手持工具（更贴近真实作业流程）
- 手持道具显示改为第一人称右下角“手部模型”样式（类似 Minecraft 的持物视图）
- 可按 `G` 丢下当前手持道具
- 在配货仓库中，手持 Reader 扫描录入后货品会自动装车到货车
- 扫过标签显示绿色，未扫标签显示红色
- 货品箱除颜色外有模型差异（水果/生鲜/烘焙/冷饮/综合）
- 改为手动搬货：选中货品按 `E` 搬起，走到车边再按 `E` 装车/卸货
- 货车可驾驶：选中货车按 `E` 上车，`W/A/S/D` 驾驶（速度高于步行），再按 `E` 下车
- 当手持 `RF Reader` 时，`E` 对货品默认优先执行扫码；丢掉Reader后`E`默认搬货
- 货品名称会显示是否“已录入系统/未录入系统”
- 人物机动性增强：`Shift` 疾跑，`Space` 跳跃
- 墙体具备体积碰撞，不可穿墙
- 第一人称下通过屏幕中心准星进行选中/放置
- 键盘操作：
  - `W/A/S/D`：第一人称行走（驾车时为前进/左转/后退/右转）
  - `Shift`：疾跑
  - `Space`：跳跃
  - `E`：交互（Reader/LCD/卡车）
  - `T`：给选中仓库货品贴 RFID 标签
  - `G`：丢下手持道具
  - 方向键：移动选中设施
  - `R`：旋转 90°
  - `F`：给选中货架补货
  - `Delete`：删除
- `Esc`：释放第一人称鼠标锁定
- `给选中货架补货`：仅对货架生效
- `保存布局` / `读取布局`：保存或恢复当前布置

## 货品类型

仓储配送货品支持多类别区分：`水果`、`生鲜`、`烘焙`、`冷饮`、`综合`，并在模型、颜色、名称和流程日志中体现。

## 当前范围

此版本是“最小可用经营仿真”：已有基础经营闭环与布局编辑能力，但仍不包含复杂员工系统、供应链、详细 UI 面板、任务系统等高级玩法。

## 与 RFID 系统联动（预留）

已暴露全局对象 `window.supermarketSim`，可在控制台或外部脚本调用：

- `window.supermarketSim.getState()`
- `window.supermarketSim.setResources({ cash, storageStock, customers, spawnInterval, salePrice })`
- `window.supermarketSim.refillAllShelves()`
- `window.supermarketSim.setOpen(true/false)`
- `window.supermarketSim.rfid.resetFlow()`
- `window.supermarketSim.rfid.registerManifest()`
- `window.supermarketSim.rfid.dispatchTruck()`
- `window.supermarketSim.rfid.deliverTruck()`
- `window.supermarketSim.rfid.confirmReceive()`
